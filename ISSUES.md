# Issues

Use this file to capture lightweight issue notes before opening or prioritizing GitHub issues.

## Template

### Title

Short, action-oriented summary.

### Type

- Bug
- Enhancement
- Documentation
- Maintenance

### Context

What is happening, where it happens, and why it matters.

### Expected Behavior

What should happen instead.

### Reproduction

1. Run:

   ```bash
   command goes here
   ```

2. Observe the result.

### Acceptance Criteria

- The expected behavior is implemented.
- Tests or verification steps are documented.
- Related documentation is updated when needed.

### Notes

Links, screenshots, logs, or follow-up questions.
